# ChatGPT 对话选择性导出工具

将 ChatGPT 对话**选择性导出**为纯静态 HTML 文件，保留原始页面样式。

> *我和 ChatGPT 的对话也是我的知识☺️，但 OpenAI 总会在不经意间剥夺你再次查看它的权利😡*

---

## ✨ 功能亮点

### 🎯 精细化选择模式
- **点击多选**：直接连续点击多个对话轮进行累加选中/取消，整轮对话（用户+ChatGPT）会一起选中
- **范围多选面板**：支持输入多个范围（如 `3-8, 12, 15`）快速选择多轮对话，可在“替换 / 追加 / 移除”间切换并实时预览
- **新版结构兼容**：支持 ChatGPT 当前的 `section[data-testid="conversation-turn-*"]` 消息结构，并过滤虚拟滚动空占位
- **快捷键**：
  - `Esc` 退出选择模式
  - `Enter` 立即导出选中内容
  - `Ctrl/Cmd + A` 全选

### 🎨 精美的侧边栏界面
- **可收起侧边栏**：位于页面右侧，不遮挡原始内容，支持一键收起/展开
- **紧凑工具面板**：选择、导出、范围选择和目录状态集中展示，减少对原页面的遮挡
- **无位移选择框**：选择高亮使用覆盖层绘制，不会撑开或挤动原始页面
- **自适应主题**：自动匹配系统的亮/暗色模式

### 💾 多种保存方式
- **浏览器下载**：直接下载到本地默认下载文件夹
- **指定目录**（Chromium 系浏览器）：设置默认保存位置，自动保存到指定文件夹
- **自动时间戳**：文件名自动添加时间戳，避免覆盖

### 📤 导出质量优化（v9.0 新增）
- **CSS 内联化**：所有样式直接内联到 HTML 文件中，离线打开样式不丢失
- **图片自包含**：对话中的图片自动转为 base64 data URI，单文件即可完整查看
- **扩展样式过滤**：自动过滤浏览器扩展（如沉浸式翻译、Grammarly）注入的无用样式
- **SVG 图标清理**：移除离线无法加载的 CDN sprite 引用
- **主题保留**：导出文件保留当前主题（亮/暗色）和 ChatGPT 主题色
- **增强降级 CSS**：即使原始 CSS 未能加载，代码块、表格、公式等仍可正常显示

### 📄 原始样式保留
- 完整保留 ChatGPT 原始页面的 CSS 样式
- 保留代码高亮、表格、图片等所有格式
- 导出文件与原始页面视觉一致

### 🔄 自愈机制
- 页面重绘后按钮自动重建，永不消失
- 基于 `MutationObserver` 的智能监听

## 📦 安装步骤

### 1. 安装浏览器扩展

安装 [Tampermonkey](https://www.tampermonkey.net/)（支持 Chrome/Edge/Brave/Vivaldi 等浏览器）

### 2. 安装脚本

1. 在 Tampermonkey 中点击 **"新建脚本"**
2. 将 [main.js](main.js) 的内容粘贴进去
3. 保存脚本（`Ctrl/Cmd + S`）

### 3. 开始使用

打开 [chatgpt.com](https://chatgpt.com)，页面右侧会出现可收起的侧边栏

## 🚀 使用方法

### 选择性导出（推荐）

1. 点击侧边栏中的 **选择模式** 按钮
2. 点击对话轮进行选择（用户+ChatGPT 回复会被一起框选）
3. 点击 **下载选中** 按钮导出

范围选择支持 `1-3, 6, 9-12`、`全部`、`奇数`、`偶数`，并会在应用前显示将影响的轮次。

![选择模式示例](docs/selection-demo.png)

### 导出全部对话

1. 直接点击 **全部导出** 按钮
2. 自动导出当前页面所有对话轮

### 设置默认保存位置（Chromium 系浏览器）

1. 点击 **设置位置** 按钮
2. 选择一个文件夹作为默认保存位置
3. 之后所有导出都会自动保存到该文件夹

> 💡 **提示**：ChatGPT 页面采用虚拟滚动，如需导出更长历史，请先向上滚动加载更多消息
> 当前脚本只导出页面已经渲染出来的消息块，空的虚拟滚动占位不会被导出。

## ❓ 常见问题

### Q1：按钮怎么不见了？
A：脚本内置自愈挂载机制，通常会自动恢复。若仍未出现：
- 检查脚本是否已启用
- 查看开发者工具（F12 → Console）是否有报错
- 刷新页面重试

### Q2：为什么导出内容不完整？
A：由于虚拟滚动机制，未滚动加载的历史消息不会被渲染。请向上滚动加载更多内容后再导出。

### Q3：Firefox 能使用目录保存功能吗？
A：Firefox 暂不支持 File System Access API，只能使用浏览器下载功能。

### Q4：为什么导出为 HTML？
A：
- HTML 保留完整样式和格式，可读性最佳
- v9.0 起所有样式和图片都内联在单个文件中，无需额外资源
- Markdown 缺乏官方标准，不同渲染器效果不一致
- JSON 可读性差，不适合直接阅读
